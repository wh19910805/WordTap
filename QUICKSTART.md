# WordTap 快速启动指南

## 安装步骤

1. **安装依赖**
```bash
npm install
```

2. **启动开发服务器**
```bash
npm run dev
```

3. **访问应用**
打开浏览器访问 `http://localhost:3000`

## 功能测试

### 1. 首页测试
- 查看数据卡片（连续打卡、词汇量、学习时长）
- 切换时间周期（今日/本周/本月/今年）
- 查看本周学习活动趋势图
- 点击"继续学习"按钮（如果有上次学习记录）

### 2. 发现页测试
- 切换到"我的课程"标签，查看已加入的课程
- 切换到"课程广场"标签，浏览所有课程
- 使用标签筛选课程
- 点击课程卡片进入详情页

### 3. 课程详情页测试
- 查看课程封面和简介
- 查看总课时数和词汇量
- 点击"加入我的课程"按钮
- 点击"开始学习"按钮进入课时列表

### 4. 课时列表页测试
- 查看所有课时
- 查看学习状态（未开始/进行中/已完成）
- 点击课时进入学习页面

### 5. 沉浸式学习页测试（核心功能）
- **VisualViewport 适配**：在移动设备或浏览器开发者工具的移动模式下测试
  - 打开软键盘，观察内容是否自动调整
  - 当前句子应保持在可视区域
  
- **打字输入**：
  - 开始输入，观察字符是否正确显示
  - 输入错误字符，观察震动反馈
  - 完成一句后，观察是否自动跳转到下一句
  
- **音频播放**：
  - 点击播放按钮播放音频
  - 点击静音按钮切换静音
  - 点击重播按钮重新播放
  
- **底部操作**：
  - 长按"提示"按钮查看提示
  - 点击"跳过"按钮跳过当前句

### 6. 设置测试
- 在个人中心或学习页面打开设置
- 切换主题（浅色/深色/跟随系统）
- 切换打字音效开关
- 切换自动播放音频开关
- 调整字体大小

## 移动端测试

### 在真实设备上测试
1. 确保开发服务器运行在局域网可访问的地址
2. 在移动设备浏览器中访问 `http://[你的IP]:3000`
3. 测试 VisualViewport 适配和键盘交互

### 使用浏览器开发者工具
1. 打开 Chrome DevTools (F12)
2. 切换到设备模拟模式 (Ctrl+Shift+M)
3. 选择移动设备（如 iPhone 12）
4. 测试键盘弹出时的视口适配

## 常见问题

### 1. 音频无法播放
- 检查音频文件路径是否正确
- 确保浏览器允许自动播放音频（某些浏览器需要用户交互后才能播放）

### 2. VisualViewport 不工作
- 确保使用支持 VisualViewport API 的浏览器（Chrome/Edge/Safari iOS 13+）
- 在移动设备或移动模拟模式下测试

### 3. 输入框无法聚焦
- 确保点击了内容区域
- 检查浏览器是否阻止了自动聚焦

### 4. 数据未保存
- 检查浏览器是否允许使用 IndexedDB
- 在浏览器开发者工具的 Application 标签中查看 IndexedDB 数据
